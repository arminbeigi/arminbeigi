# فاز ۳ — ماژول ۳: محصولات HVAC (Products)

> وضعیت: **اجراشده و اعتبارسنجی‌شده.** build سبز، ۲۲ تست واحد سبز،
> و ۱۷ سنجش end-to-end روی اپ واقعی + PostgreSQL واقعی سبز.

## ۱) چه ساخته شد
کاتالوگ محصولات HVAC با برند:
- CRUD محصول برای دسته‌های **دیگ/مشعل/پمپ/منبع/رادیاتور/متعلقات** با مشخصات فنی صریح
  (ظرفیت kcal/kW، راندمان، فشار، دبی، ارتفاع آبدهی، حجم لیتر، جنس، تعداد پره) + `specs` آزاد (JSON).
- مدیریت **برند** (`Brand`) با نام لاتین/فارسی.
- فهرست با **فیلتر دسته/سوخت/نوع‌دیگ/برند/فعال** + صفحه‌بندی.
- **جست‌وجوی فازی** روی نام (فارسی، `fa_normalize`+trigram) و SKU (لاتین، trigram).
- قیمت به‌صورت `Decimal` (ریال) با خروجی رشته‌ای برای حفظ دقت.

## ۲) Endpointها (تحت RBAC)
| متد | مسیر | مجوز |
|---|---|---|
| POST | `/api/products/brands` | `products:write` |
| GET | `/api/products/brands` | `products:read` |
| POST | `/api/products` | `products:write` |
| GET | `/api/products` (فیلتر + `q`) | `products:read` |
| GET | `/api/products/:id` | `products:read` |
| PATCH | `/api/products/:id` | `products:write` |
| DELETE | `/api/products/:id` | `products:delete` |

## ۳) تصمیم‌های کلیدی
- **مشخصات فنی** ستون واقعی (برای فیلتر/ایندکس) + `specs` JSON برای داده‌ی منعطف.
- **جست‌وجوی فازی دوزبانه**: `fa_normalize("name") % fa_normalize(q) OR "sku" % q OR "sku" ILIKE %q%`،
  مرتب با `GREATEST(similarity(...))` — برندهای لاتین (Buderus) و نام فارسی هر دو پیدا می‌شوند.
- **Decimal**: ورودی به‌صورت `number`، خروجی `priceIrr` رشته (`Decimal.toString()`) تا دقت ریالی حفظ شود.
- خطای یکتایی SKU/برند (P2002) → `409`؛ برند ناموجود (P2025) → پیام مناسب.

## ۴) شواهد اعتبارسنجی
- `nest build` ⇒ بدون خطا.
- `jest` ⇒ **۲۲/۲۲** (۴ تست جدید: connect برند، Decimal→رشته، مسیر فازی/فهرست، NotFound، Conflict).
- اجرای واقعی + **۱۷ سنجش HTTP**: ساخت برند و دیگ چدنی (قیمت `250000000` رشته، برند «بودروس»)،
  پمپ/رادیاتور، **SKU تکراری ۴۰۹**، فیلتر دسته/سوخت، **فازی «بودروس چدنی»** و **فازی SKU «PMP»**،
  PATCH قیمت/موجودی، حذف + ۴۰۴، RBAC (viewer ۴۰۳)، و enum نامعتبر ۴۰۰.

## ۵) Impact Analysis
- ایزوله در `modules/products` + یک خط در `app.module.ts`.
- آماده برای ماژول‌های بعد: اقلام معامله (`DealItem`) و اقلام پروژه (`ProjectItem`) به این کاتالوگ
  ارجاع می‌دهند؛ موتور پیشنهاد محصول فاز ۶ روی همین مشخصات فنی کار می‌کند.
- خارج از این ماژول برای بعد: دسته‌بندی سلسله‌مراتبی، تصاویر محصول، سینک ووکامرس (`externalRef`).
