# فاز ۵ — فرانت‌اند (Next.js فارسی/RTL)

> ساخت تدریجی در واحدهای قابل‌تحویل.

## واحد ۱ — پایه + لاگین + داشبورد + پوسته‌ی RTL ✅
> `next build` سبز (۱۷ مسیر)، و **۱۵ سنجش مرورگر** با Playwright سبز.

- زیرساخت Next.js ۱۴ (App Router) + TypeScript + Tailwind، کاملاً **RTL** با **Vazirmatn** (`@fontsource`).
- تم صنعتی HVAC (steel + flame)، احراز هویت سمت کلاینت با **تازه‌سازی خودکار توکن** و محافظت از مسیر.
- صفحه‌ی ورود فارسی، پوسته‌ی داشبورد با **منوی کامل فارسی** (فیلتر بر اساس مجوز) و تاپ‌بار.
- داشبورد با **داده‌ی واقعی API** (کارت‌های آماری + تماس‌های اخیر، تاریخ شمسی، اعداد فارسی).
- صفحات Placeholder برای بقیه‌ی منو.
- **باگ مسیر رفع‌شده**: گروه `(dashboard)` سگمنت URL نمی‌ساخت → بازساختار به سگمنت واقعی `dashboard/`.

## واحد ۲ — صفحه‌ی مشتریان ✅
> وضعیت: **اجراشده و اعتبارسنجی‌شده.** `next build` سبز، و **۶ سنجش مرورگر** Playwright روی اپ واقعی + API + PostgreSQL سبز.

### چه ساخته شد
- **فهرست مشتریان**: جست‌وجوی زنده (debounce) با **جست‌وجوی فازی فارسی**، فیلتر نوع/وضعیت (چیپ)،
  جدول با نام/نوع/وضعیت/شماره اصلی/امتیاز سرنخ/تاریخ، صفحه‌بندی، دکمه‌ی افزودن.
- **مودال ساخت مشتری**: انتخاب نوع، نام، نام شرکت (برای B2B)، شماره‌های چندگانه.
- **صفحه‌ی جزئیات** (`/dashboard/customers/[id]`): پروفایل، **تغییر وضعیت** درجا، **مدیریت شماره** (افزودن/حذف)،
  **ویرایش** (مودال)، **حذف** (با تأیید)، و **تماس‌های اخیر همان مشتری**.
- کامپوننت‌های قابل‌استفاده‌ی مجدد: Badge، Modal، Pagination، Loading/Empty/Error، و هوک `useDebounce`.
- برچسب‌های فارسی enumها در `lib/enums.ts` با رنگ نشان.
- کنترل دسترسی UI: دکمه‌های نوشتن/حذف فقط با مجوز (`customers:write/delete`).

  افزودن شماره دوم، **جست‌وجوی فازی «تاسیسات اریا» → «آریا»**، و فیلتر نوع «پیمانکار»).

## واحد ۳ — مرکز تماس + لاگ تماس‌ها ✅
> وضعیت: **اجراشده و اعتبارسنجی‌شده.** `next build` سبز، و **۸ سنجش مرورگر** Playwright روی اپ واقعی + API + PostgreSQL سبز.

### چه ساخته شد
- **کنسول مرکز تماس** (`/dashboard/call-center`):
  - **شماره‌گیر کلیک‌تو‌کال** (keypad فارسی) → `POST /telephony/originate` با داخلی کاربر جاری.
  - **بنر تماس ورودی زنده** با نام مشتری و دکمه‌ی «مشاهده پرونده مشتری».
  - **فید تماس‌های زنده** با **به‌روزرسانی خودکار** (polling هر ۴ ثانیه؛ در فاز ۷ با WebSocket جایگزین می‌شود).
  - **وضعیت خط** (متصل/قطع • شبیه‌ساز/Issabel).
  - **شبیه‌سازی تماس ورودی** (فقط حالت Mock) برای آزمون/دمو.
  - کمترین کلیک: یک ورودی + یک دکمه برای تماس؛ کلیک روی نام مشتری → پرونده.
- **لاگ تماس‌ها** (`/dashboard/calls`): فهرست با فیلتر جهت/وضعیت + صفحه‌بندی، با لینک به مشتری.
- کامپوننت مشترک `CallRow` و نگاشت رنگ وضعیت تماس.

### شواهد اعتبارسنجی
- `next build` ⇒ بدون خطا (call-center + calls، type-check کامل).
- **Playwright روی Chromium واقعی** + اپ + API + PostgreSQL: **۸/۸**
  (بارگذاری کنسول، وضعیت متصل/شبیه‌ساز، **شبیه‌سازی تماس ورودی → ظاهرشدن در فید زنده با سرنخ خودکار**،
  **کلیک‌تو‌کال → «تماس برقرار شد»**، لاگ تماس‌ها، و فیلتر جهت).
- اسکرین‌شات کنسول بازبینی شد.

## واحد ۴ — پایپ‌لاین فروش (کانبان) ✅
> وضعیت: **اجراشده و اعتبارسنجی‌شده.** `next build` سبز، و **۹ سنجش مرورگر** Playwright سبز.

### چه ساخته شد
- **تخته‌ی کانبان** (`/dashboard/deals`): ستون‌ها = مراحل پایپ‌لاین (از `GET /deals/pipelines`)،
  کارت معامله با عنوان/مشتری/مبلغ (تومان)/کارشناس، و **مجموع مبلغ هر ستون**.
- **جابه‌جایی معامله**: کشیدن‌ورهاکردن (HTML5 DnD) + **منوی انتقال** روی هر کارت (قابل‌دسترس/آزمون‌پذیر).
- **بستن برنده/بازنده داده‌محور**: انتقال به مرحله‌ی برنده → WON؛ به مرحله‌ی بازنده → **مودال دلیل** → LOST.
- **مودال افزودن معامله** با **انتخاب مشتری جست‌وجویی** (`CustomerSelect` قابل‌استفاده‌ی مجدد) و مبلغ تومانی.

### دو باگ واقعی کشف‌شده از طریق UI و رفع‌شده
1. **جست‌وجوی عبارت کوتاه**: similarity تریگرامِ یک عبارت کوتاه («آریا») در برابر نام بلند زیر آستانه می‌افتاد و
   نتیجه نمی‌داد. در بک‌اند، علاوه بر `%` تریگرام، **`ILIKE` زیررشته‌ای روی `fa_normalize`** اضافه شد
   (همان ایندکس GIN تریگرام شتابش می‌دهد). آزمون: «آریا» قبلاً ۰ نتیجه → اکنون «شرکت تأسیساتی آریا».
2. **`limit=200` در کوئری تخته**: `PaginationDto` سقف ۱۰۰ دارد؛ درخواست ۲۰۰ خطای **۴۰۰** می‌داد و تخته خالی می‌ماند.
   کلاینت به `limit=100` اصلاح شد.

### شواهد اعتبارسنجی
- `next build` سبز؛ `nest build` سبز (پس از اصلاح بک‌اند).
- **Playwright روی Chromium واقعی** + اپ + API + PostgreSQL: **۹/۹**
  (۶ ستون مرحله، **افزودن معامله با انتخاب مشتری**، **انتقال با منو به ارزیابی‌شده**،
  **بستن به‌عنوان ناموفق با دلیل**).
- اسکرین‌شات تخته بازبینی شد.
