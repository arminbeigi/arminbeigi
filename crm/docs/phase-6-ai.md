# فاز ۶ — لایه‌ی هوش مصنوعی (AI)

> وضعیت: **اجراشده و اعتبارسنجی‌شده (حالت Mock).** build سبز، ۵۵ تست واحد سبز،
> و ۱۶ سنجش end-to-end روی اپ واقعی + PostgreSQL واقعی سبز.
> طبق تصمیم فاز ۰: مدل خودمیزبان + Whisper خودمیزبان، با اینترفیس قابل‌تعویض، شروع با Mock.

## ۱) چه ساخته شد
لایه‌ی AI کامل پشت اینترفیس‌های قابل‌تعویض:
- **ارائه‌دهنده‌ی LLM**: `MockLlmProvider` (خروجی فارسی قطعی) + `OpenAiLlmProvider` (سرور سازگار با OpenAI، مدل خودمیزبان).
- **ارائه‌دهنده‌ی STT**: `MockSttProvider` (متن HVAC واقع‌نما) + `WhisperSttProvider` (Whisper خودمیزبان، `whisper-asr-webservice`).
- **تشخیص نیت فارسی** (`intent.ts`): کلیدواژه‌محور و **واقعی** (نه فقط Mock) — خرید/خرابی/نصب/سرویس، مقاوم به املای عربی (ي/ك).
- **خط لوله‌ی تحلیل تماس** (`processCall`): رونویسی (در نبود متن) → خلاصه → نیت → امتیاز سرنخ → پیشنهاد محصول؛
  ذخیره‌ی همه در `AIInsight` و به‌روزرسانی `Call.intent/transcript` و `Customer.leadScore`.
- **موتور پیشنهاد محصول**: استخراج دسته از کلیدواژه‌های متن (پکیج/دیگ→BOILER، پمپ→PUMP، …) و پیشنهاد محصولات فعال.
- **امتیازدهی سرنخ**: قاعده‌محور بر اساس نیت + پاسخ‌دهی تماس + وجود معامله‌ی باز (۰..۱۰۰).
- **دستیار فارسی** (`assistant`): پرسش‌های متداول (تعداد مشتری/معاملات باز/تماس امروز/پروژه‌ها) را مستقیم از داده پاسخ می‌دهد؛ بقیه به LLM با زمینه‌ی آماری می‌رود.

## ۳) معماری قابل‌تعویض
```
Call.recordingPath ─▶ STT (Mock|Whisper) ─▶ transcript ─▶ LLM (Mock|OpenAI-compat) ─▶ summary
                                                        └▶ detectIntent (کلیدواژه فارسی) ─▶ intent
                                                        └▶ recommendFromText (کاتالوگ) ─▶ پیشنهاد
                                                        └▶ scoreLead (قاعده‌محور) ─▶ Customer.leadScore
  انتخاب با AI_MOCK / STT_MOCK. هنگام راه‌اندازی مدل/Whisper خودمیزبان:
  AI_MOCK=false + AI_BASE_URL/AI_MODEL، STT_MOCK=false + STT_BASE_URL — بدون تغییر منطق.
```

## ۴) شواهد اعتبارسنجی
- `nest build` ⇒ بدون خطا.
- `jest` ⇒ **۵۵/۵۵** (۷ تست جدید: تشخیص نیت ۴ کلاس + املای عربی، scoreLead، recommend، processCall کامل/خالی، assistant).
- اجرای واقعی + **۱۶ سنجش HTTP**: status mock، ساخت تماس با ضبط، **تحلیل کامل** (رونویسی، خلاصه «موضوع: …»،
  **نیت PURCHASE**، **امتیاز سرنخ ۸۵**، ۳ پیشنهاد محصول)، ذخیره‌ی ۳ نوع بینش، ثبت leadScore روی مشتری،
  پیشنهاد برای مشتری، **دستیار فارسی** (تعداد مشتری؛ «۲ معامله‌ی باز با ارزش ۵۰۰٬۰۰۰٬۰۰۰ تومان»)، و RBAC (viewer ۴۰۳).

## ۵) یک باگ واقعی کشف‌شده و رفع‌شده
دستیار برای «معاملات» (جمع) پاسخ نمی‌داد چون زیررشته‌ی «معامله» در «معاملات» نیست؛ به ریشه‌ی «معامل» تغییر یافت.

## ۶) راه‌اندازی مدل واقعی (هنگام آماده‌شدن)
- LLM: یک سرور سازگار با OpenAI بالا بیاورید (vLLM/Ollama)، سپس `AI_MOCK=false`, `AI_BASE_URL`, `AI_MODEL`.
- STT: سرویس `whisper-asr-webservice` (مدل فارسی)، سپس `STT_MOCK=false`, `STT_BASE_URL`.
- هیچ منطق CRM تغییر نمی‌کند؛ فقط ارائه‌دهنده عوض می‌شود.
