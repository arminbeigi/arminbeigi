# فاز ۳ — ماژول ۴: پروژه‌ها (Projects)

> وضعیت: **اجراشده و اعتبارسنجی‌شده.** build سبز، ۲۸ تست واحد سبز،
> و ۱۹ سنجش end-to-end روی اپ واقعی + PostgreSQL واقعی سبز.

## ۱) چه ساخته شد
مدیریت پروژه‌های تأسیساتی:
- CRUD پروژه برای انواع **موتورخانه/پکیج/دیگ چدنی‑فولادی/نصب/سرویس** با فیلدهای فنی
  (متراژ، طبقات، واحد، بار حرارتی) و برآورد/هزینه نهایی (Decimal ریال).
- **اقلام پروژه** (`ProjectItem`) با ارجاع اختیاری به کاتالوگ محصولات.
- **گردش‌کار وضعیت** با اعتبارسنجی انتقال (state machine).
- اتصال به مشتری (الزامی)، آدرس محل اجرا، و مدیر پروژه.
- فهرست با فیلتر نوع/وضعیت/مشتری/مدیر + جست‌وجوی فازی عنوان.

## ۲) گردش‌کار وضعیت (State Machine)
```
DRAFT → SURVEY → PROPOSAL → IN_PROGRESS → COMPLETED
  │        │          │           ↑↓
  └────────┴──────────┴──────→ ON_HOLD
  همه (به‌جز پایانی‌ها) → CANCELLED
COMPLETED و CANCELLED پایانی‌اند.
```
انتقال نامجاز ⇒ `400`؛ ورود به `COMPLETED` به‌طور خودکار `completedAt` را ست می‌کند.

## ۴) شواهد اعتبارسنجی
- `nest build` ⇒ بدون خطا.
- `jest` ⇒ **۲۸/۲۸** (۶ تست جدید: connect مشتری/اقلام، انتقال مجاز/نامجاز، completedAt خودکار، فازی/فهرست، NotFound).
- اجرای واقعی + **۱۹ سنجش HTTP**: ساخت موتورخانه با اقلام و فیلدهای فنی (برآورد `8500000000`)،
  قلم با نام محصول، **گردش DRAFT→SURVEY→PROPOSAL→IN_PROGRESS**، **انتقال نامجاز ۴۰۰**،
  COMPLETED + completedAt خودکار + finalIrr، فیلتر نوع/مشتری، **فازی «موتورخانه ارین»**،
  افزودن/حذف قلم، مشتری ناموجود ۴۰۴، RBAC viewer ۴۰۳، enum ۴۰۰.
